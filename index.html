FastAPI - Token Dependency

Concept

This example shows how to protect a FastAPI route using a custom dependency that checks if a token is correct before allowing access. If the token is valid, the request proceeds; otherwise, an error is returned.

Python Code

from fastapi import FastAPI, Depends, HTTPException
import uvicorn

app = FastAPI()

# ✅ Dependency that checks for a valid token
def check_token(token: str):
    def check():
        if token == "123":
            return "Authorized"
        else:
            raise HTTPException(status_code=400, detail="notauthorized")
    return check

# ✅ Route with token-based access
@app.get("/get_detail")
def get_web(auth: str = Depends(check_token("124"))):
    return {"message": "Hi", "auth_status": auth}

if __name__ == "__main__":
    uvicorn.run(app)

Sample Output

GET /get_detail

Response:
{
  "detail": "notauthorized"
}

Explanation

  • FastAPI(): Creates the app instance.
  • check_token(token): A dependency factory that returns a nested function.
  • check(): Validates whether the token is "123". If yes, it returns "Authorized".
  • Depends(...): Injects the result of the dependency into the route function.
  • "124" passed to check_token: Incorrect, so it raises a 400 error.
  • uvicorn.run(...): Starts the FastAPI app.
Note:

To authorize successfully, change Depends(check_token("124")) to Depends(check_token("123")).
